· 痛点分析

传统招聘的四大困境

制造业企业招聘流程复杂、环节多，传统方式面临效率低下、体验割裂等突出问题

⏱️

流程冗长耗时

从JD编写、渠道发布、简历筛选、面试安排到录用通知，全流程人工操作，一个岗位平均耗时2-3周

📊

筛选效率低下

HR每天浏览数百份简历，人工匹配度判断标准不统一，优质候选人容易被遗漏

🤖

面试资源瓶颈

大规模招聘时，面试官时间有限，候选人等待周期长，面试标准难以一致

🔗

信息孤岛严重

JD发布、简历收集、面试安排、评估记录分散在不同系统，数据无法打通

· 解决方案

AI Agent 驱动的全流程自动化

将招聘拆解为7个标准步骤，每个步骤由专属AI Agent负责，用户只需确认即可推进

1

📄 JD 智能生成与发布

输入岗位名称和关键要求，AI自动生成完整的岗位描述和任职要求，一键发布到招聘网站

JD Generator Agent JD Publisher Agent
2

📥 简历智能采集与去重

自动从后台获取新增投递，只处理待审核简历，筛选后自动标记状态避免重复处理，淘汰简历可批量清理

Resume Collector Agent
3

🔍 AI 简历匹配筛选

解析简历内容，与岗位JD进行多维度匹配度分析（技能匹配、经验匹配、学历匹配），输出匹配分数和亮点分析

Resume Parser Agent Match Evaluator
4

📧 面试邀请与时间确认

初筛通过后自动发送面试邀请邮件，候选人通过链接自助选择面试时段，系统自动确认

HR Boot Agent
5

🤖 AI 数字人面试

创建AI面试房间，支持数字人虚拟面试官，按预设题库自动提问，实时评估候选人回答

Interviewer Agent
6

📊 综合评估与报告

简历匹配评分（30%）+ 面试评分（70%）加权计算，生成优劣势分析、风险提示和录用建议

Evaluator Agent
7

📬 智能通知发送

通过/待定/不通过三类结果分别发送个性化通知邮件，通知HR负责人汇总结果

Notifier Agent

· 核心亮点

四大创新设计

让招聘从"人工驱动"转变为"AI驱动"，人只做关键决策

⚙️

状态机驱动流程

13个状态节点 + 13个转换事件，精确控制招聘流程的每一步

  • 每个状态有明确的进入条件和操作
  • 支持自动转换和手动干预
  • 完整的状态变更历史记录
  • 可扩展的守卫条件和钩子系统
🤖

多Agent协作体系

每个招聘环节由专业Agent负责，职责清晰、协作有序

  • JD生成器 → JD发布器 → 简历采集器
  • 简历解析器 → 面试官Agent → 评估器
  • 支持实时流式状态反馈
  • 用户仅需确认关键决策点
🔌

MCP 标准化协议

通过 Model Context Protocol 实现AI与系统的标准化互通

  • 10+ 标准MCP工具接口
  • AI可调用：发布岗位、分析简历、创建面试
  • 资源统一管理（岗位、候选人数据）
  • 天然支持Agent与工具的解耦
🔄

全自动面试管道

从邀请发送到结果获取，面试流程全自动完成

  • 筛选通过 → 自动发送邀请邮件
  • 候选人选择时间 → 自动创建面试房间
  • 面试前5分钟 → 自动启动面试
  • 面试完成 → 自动获取评分与评估
